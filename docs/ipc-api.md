# IPC API Reference

All communication between the renderer and the main process goes through the `window.api` bridge exposed by the preload script. This document lists every channel, its direction, inputs, and return type.

Channels use `domain:action` naming. Request/response channels use `ipcMain.handle` / `ipcRenderer.invoke`. Push events use `ipcRenderer.on` / `mainWindow.webContents.send`.

---

## `sessions`

### `sessions:list`

Load the list of all Claude sessions from `~/.claude/projects/`.

**Direction:** renderer â†’ main (request/response)

```ts
window.api.sessions.list(dir?: string): Promise<SessionInfo[]>
```

| Parameter | Type | Description |
|---|---|---|
| `dir` | `string?` | If provided, filter sessions to those whose `cwd` matches this path |

**Returns:** Array of `SessionInfo` objects sorted by last-modified descending.

```ts
interface SessionInfo {
  sessionId: string;
  cwd: string;
  lastModified: number;   // Unix ms timestamp
  gitBranch?: string;
  summary?: string;       // Last user prompt (truncated to 120 chars)
  firstPrompt?: string;   // First user prompt (truncated to 120 chars)
}
```

---

### `sessions:getMessages`

Load all user and assistant messages for a session.

**Direction:** renderer â†’ main (request/response)

```ts
window.api.sessions.getMessages(sessionId: string, dir?: string): Promise<unknown[]>
```

| Parameter | Type | Description |
|---|---|---|
| `sessionId` | `string` | The session ID (matches the JSONL filename) |
| `dir` | `string?` | Project directory, used to narrow the file search |

**Returns:** Array of raw JSONL message objects (`type: "user"` or `type: "assistant"`). The renderer's `parse-turns.ts` converts these into structured `Turn[]`.

---

### `sessions:getSubagentMessages`

Load messages for a subagent run that was spawned by an `Agent` / `Task` tool call.

**Direction:** renderer â†’ main (request/response)

```ts
window.api.sessions.getSubagentMessages(
  sessionId: string,
  toolUseId: string,
  dir?: string
): Promise<unknown[]>
```

| Parameter | Type | Description |
|---|---|---|
| `sessionId` | `string` | Parent session ID |
| `toolUseId` | `string` | The `id` of the `Agent`/`Task` tool use block in the parent session |
| `dir` | `string?` | Project directory (required â€” returns `[]` without it) |

**Returns:** Array of raw JSONL message objects from the subagent's own JSONL file. The lookup strategy: find the `promptId` in the parent session that has a `tool_result` matching `toolUseId`, then find the subagent file in `~/.claude/projects/<slug>/<sessionId>/subagents/` whose first line has that `promptId`.

## `claude`

### `claude:start`

Start a new Claude session or resume an existing one.

**Direction:** renderer â†’ main (request/response)

```ts
window.api.claude.start(prompt: string, options: Record<string, unknown>): Promise<void>
```

| Option | Type | Description |
|---|---|---|
| `cwd` | `string` | Working directory for the `claude` process |
| `model` | `string` | Model to use (`"sonnet"`, `"opus"`, `"haiku"`) |
| `effort` | `string` | Thinking effort (`"low"`, `"medium"`, `"high"`) |
| `resume` | `string?` | Session ID to resume (omit for a new session) |
| `correlationId` | `string` | UUID generated by the renderer to correlate streaming messages before a `sessionId` is known |

The call resolves when the `claude` process exits. Streaming output arrives via `claude:message` events while the process is running.

---

### `claude:cancel`

Send `SIGTERM` to a running `claude` process.

**Direction:** renderer â†’ main (request/response)

```ts
window.api.claude.cancel(sessionId?: string): Promise<void>
```

| Parameter | Type | Description |
|---|---|---|
| `sessionId` | `string?` | If provided, cancel only the session with this ID. If omitted, cancel all active sessions. |

### `claude:message` *(push event)*

Fired for each JSONL line written to `claude`'s stdout.

**Direction:** main â†’ renderer (push)

```ts
window.api.claude.onMessage(callback: (message: unknown) => void): () => void
```

The callback receives the parsed JSONL object with an additional `correlationId` field injected by the main process. Returns an unsubscribe function.

Common `message.type` values:
- `"system"` with `subtype: "init"` â€” contains `session_id`
- `"assistant"` â€” contains the assistant's content blocks (text, tool use, thinking)
- `"user"` â€” echoes the user turn or tool results
- `"result"` â€” final result message with token counts and cost

---

### `claude:error` *(push event)*

Fired when the `claude` process exits with a non-zero code or fails to start.

**Direction:** main â†’ renderer (push)

```ts
window.api.claude.onError(callback: (error: unknown) => void): () => void
```

The callback receives `{ message: string, correlationId: string }`.

---

### `claude:done` *(push event)*

Fired when the `claude` process exits (regardless of exit code).

**Direction:** main â†’ renderer (push)

```ts
window.api.claude.onDone(
  callback: (event: { correlationId: string; sessionId: string | null }) => void
): () => void
```

`sessionId` is `null` if the process exited before a `system:init` message was received.

## `files`

### `files:readDir`

Read the immediate children of a directory for the file explorer.

**Direction:** renderer â†’ main (request/response)

```ts
window.api.files.readDir(dirPath: string): Promise<FileTreeNode[]>
```

**Returns:** Sorted array of `FileTreeNode` objects (directories first, then files, both alphabetical). Filtered to exclude `node_modules`, `.git`, `dist`, `build`, `.DS_Store`, and other hidden files.

```ts
interface FileTreeNode {
  name: string;
  path: string;         // Absolute path
  isDirectory: boolean;
  extension?: string;   // File extension including dot (e.g. ".ts"), undefined for directories
  children?: FileTreeNode[];
}
```

---

### `files:readFile`

Read the text content of a file.

**Direction:** renderer â†’ main (request/response)

```ts
window.api.files.readFile(filePath: string): Promise<string>
```

Files over 512 KB return the string `"[File too large to preview]"` instead of content.

## `permission`

### `permission:request` *(push event)*

Fired when the `claude` CLI makes a `PreToolUse` hook call for a tool that is not on the always-allowed list. The renderer should display a permission card and wait for the user to respond.

**Direction:** main â†’ renderer (push)

```ts
window.api.permission.onRequest(callback: (request: PermissionRequest) => void): () => void
```

```ts
interface PermissionRequest {
  id: string;                          // Request ID â€” pass back to permission.respond()
  sessionId: string;
  toolName: string;
  toolUseId: string;
  toolInput: Record<string, unknown>;
}
```

---

### `permission:autoApproved` *(push event)*

Fired when a tool was automatically approved from the always-allowed list. The renderer shows a subtle notification in the chat stream.

**Direction:** main â†’ renderer (push)

```ts
window.api.permission.onAutoApproved(callback: (event: AutoApprovedEvent) => void): () => void
```

```ts
interface AutoApprovedEvent {
  toolName: string;
  toolUseId: string;
  sessionId: string;
  toolInput: Record<string, unknown>;
}
```

---

### `permission:respond`

Send the user's allow/deny decision back to the main process. This unblocks the pending `PreToolUse` HTTP request and lets the CLI continue.

**Direction:** renderer â†’ main (fire-and-forget, no response)

```ts
window.api.permission.respond(id: string, decision: "allow" | "deny"): void
```

Note: this uses `ipcRenderer.send` (one-way), not `invoke`.

---

### `permission:alwaysAllow`

Add a tool to the always-allowed list. Persisted across restarts via `electron-store`. Future calls to that tool will be auto-approved without a prompt.

**Direction:** renderer â†’ main (request/response)

```ts
window.api.permission.alwaysAllow(toolName: string): Promise<void>
```

## Shared Types

Defined in `src/shared/types.ts` and used across the IPC boundary.

```ts
type ToolName =
  | "Read" | "Bash" | "WebSearch" | "Edit" | "Write"
  | "Glob" | "WebFetch" | "Grep" | "Agent" | "Task"
  | "TaskCreate" | "TaskUpdate";

interface Turn {
  number: number;
  timestamp: string;
  userMessage: TurnUserMessage | null;
  assistantBlocks: AssistantBlock[];
  model?: string;
  inputTokens?: number;
  outputTokens?: number;
  durationMs?: number;
  costUsd?: number;
}

type AssistantBlock = TextBlock | ThinkingBlock | ToolUseBlock;

interface ToolUseBlock {
  type: "tool_use";
  id: string;
  name: string;
  input: Record<string, unknown>;
  result?: ToolResult;
}

interface ToolResult {
  content: string;
  isError: boolean;
  durationMs?: number;
  bytes?: number;
}
```
